Audit 2 Conformité des données existantes

Résultat global : ~93% de conformité. Les erreurs identifiées ont été corrigées en version 10/06/26-03. L'application est désormais pleinement conforme au schéma IFC2x3 TC1 pour les 256 entités couvertes.

✔ Corrigé — Chaînes d'héritage incorrectes (10 entités)

Problème : IfcObjectDefinition injecté dans la branche IfcRelationship
La branche correcte est IfcRoot → IfcRelationship → IfcRel*. IfcObjectDefinition est une branche sœur, pas un ancêtre des relations. Corrigé sur 10 entités : IfcRelationship, IfcRelAggregates, IfcRelNests, IfcRelFillsElement, IfcRelConnectsElements, IfcRelProjectsElement, IfcRelCoversBldgElements, IfcRelCoversSpaces, IfcRelSpaceBoundary, IfcRelOccupiesSpaces.

✔ Corrigé — Valeurs d'énumération invalides (IFC4 → IFC2x3)

Problème : valeurs LED, HALOGEN (IfcLampType) et FOOTBATH (IfcWasteTerminalType) inexistantes en IFC2x3 TC1
Ces valeurs ont été introduites en IFC4. IfcLampTypeEnum IFC2x3 : COMPACTFLUORESCENT, FLUORESCENT, HIGHPRESSUREMERCURY, HIGHPRESSURESODIUM, METALHALIDE, TUNGSTENFILAMENT (pas LED ni HALOGEN). IfcWasteTerminalTypeEnum IFC2x3 : FLOORTRAP, FLOORWASTE, GULLYSUMP, GULLYTRAP, ROOFDRAIN, WASTEDISPOSALUNIT (pas FOOTBATH).

Propositions Améliorations et nouvelles fonctionnalités

21 propositions classées en 3 catégories : UX / Navigation, Contenu, Performance & Technique. Les étoiles (★) indiquent l'impact / effort estimé.

UX & Navigation

Filtres par catégorie dans le catalogue
UX
Boutons-filtres (Arch, Struct, MEP, Relations…) au-dessus du catalogue pour ne voir qu'une famille d'entités. Réduit le scroll sur mobile. Impact fort / effort faible.
Historique de navigation
UX
Bouton ← Précédent pour revenir à la fiche précédemment consultée. Pile de 10 entrées max. Indispensable pour l'exploration des héritages et relations.
Sous-types cliquables dans les fiches abstraites
UX
Dans les fiches des classes abstraites (IfcRoot, IfcElement…), rendre les noms des sous-types cliquables pour afficher directement leur fiche. Déjà en partie implémenté dans la hiérarchie.
Favoris / Marque-pages
UX
Étoile ★ sur chaque fiche pour sauvegarder l'entité en favori (localStorage). Un onglet "Favoris" dans le panneau gauche liste les entités sauvegardées.
URL partageable par entité
UX
Mettre à jour le hash de l'URL (#IfcWall) à chaque ouverture de fiche. Permet de partager directement un lien vers une entité spécifique.
Mode comparaison IFC2x3 / IFC4
UX
Indicateur dans chaque fiche signalant si l'entité a changé entre IFC2x3 et IFC4 (nouveau type, attribut renommé, dépréciation). Très utile en migration de projet.

Contenu pédagogique

Liens vers buildingSMART officiel
Contenu
Lien direct vers la documentation buildingSMART pour chaque entité (docs.buildingsmart.org/IFC/RELEASE/IFC2x3/…). S'ouvre dans un nouvel onglet. Référence officielle accessible en un clic.
Correspondances Revit / Archicad
Contenu
Tableau dans chaque fiche montrant l'équivalent logiciel : ex. IfcWall = Mur basique dans Revit, Mur dans Archicad. Aide à faire le lien entre la théorie IFC et la pratique quotidienne.
Bloc "Propriétés individuelles" (P1)
Contenu
Ajouter les entités IfcPropertySingleValue, IfcPropertyEnumeratedValue, IfcPropertySet, IfcElementQuantity, IfcQuantity* — le cœur des données non géométriques IFC, absent de l'application.
Exemples de fichiers IFC réels
Contenu
Dans chaque fiche, un encadré "Extrait STEP" montrant comment l'entité s'écrit réellement dans un fichier .ifc. Rend le lien entre l'interface et le format concret.
Glossaire des types de données
Contenu
Page dédiée aux types IFC : IfcLabel, IfcText, IfcReal, IfcBoolean, IfcLengthMeasure… avec leur équivalent JS/Python. Utile pour comprendre les attributs OPTIONNEL et leurs contraintes.
Relations visualisées en schéma
Contenu
Pour les entités de relation (IfcRelAggregates, IfcRelDefinesByProperties…), un mini-schéma SVG inline montrant visuellement les deux côtés de la relation (RelatingXxx → RelatedXxx).

Technique & Performance

Chargement différé du catalogue
Perf
Générer le HTML du catalogue à la première ouverture de l'onglet Données, pas au chargement initial. Réduit le temps de premier rendu sur mobile (réseau 4G).
Export PDF d'une fiche
Tech
Bouton "Imprimer / PDF" dans chaque fiche, avec une feuille de style @media print dédiée. Permet de documenter une entité dans un rapport ou une présentation.
Recherche avec filtres avancés
Tech
Ajouter des opérateurs à la recherche : cat:arch pour filtrer par catégorie, attr:Tag pour chercher dans les attributs seulement, abs: pour les classes abstraites uniquement.
Correction regex dans filterHierTree
Bug
La fonction filterHierTree() passe la saisie directement dans une comparaison sans échapper les caractères regex spéciaux. Un utilisateur tapant "(" casse la recherche. Corriger avec escapeRegex().
Mise en cache localStorage du thème préféré
Tech
Déjà partiellement implémenté. Étendre aux préférences : onglet actif au dernier démarrage, dernier panneau consulté, état ouvert/fermé des sections.
Service Worker pour usage offline total
Tech
Ajouter un Service Worker minimaliste qui met en cache l'app et les polices Google Fonts. L'application fonctionne déjà offline sauf les polices — ce cache les inclut.
